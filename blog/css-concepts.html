What are the differences between an ID selector and a class selector?

CSS allows us to create rules on how to change the appearance of our webpages. A CSS rule consists of a selector and a declaration. The selector determines what scope of HTML elements a declaration will apply to, while a declaration specifies what properties and values each selected element will take on. An ID selector applies to any element that has a matching ID attribute. A class selector applies to any element that has a matching class attribute. In short, IDs are unique while classes are generic!

When should I use an ID selector?

Let's first recognize two important characteristics for IDs. First, an element may only have one ID attribute. Second, each page can only have one instance of a particular ID. Our HTML code will fail validation if we use an ID more than once on a page. An ID selector should be used to style a unique element. Here is an example for using an ID selector to style the navigation bar of this webpage.

#menu {
  padding: 0;
  text-align: center;
}
#menu li {
  display: inline;
}
#menu li a {
  background-color: #1EFFC7;
  color: navy;
  padding: 10px 20px;
  text-decoration: none;
  border-radius: 4px;
  font-weight: bold;
}
#menu li a:hover {
    background-color: #FF2442;
}

When should I use a class selector?

In contrast, a class selector is meant for use among one to many elements. An element may also have multiple class attributes. Classes are an efficient way to style multiple elements at once regardless of the element type. Assuming a webpage has multiple div elements, we can style these together or separately using classes. Here is an example for using class selectors to style the following code block. Two different classes are being used to style a div element. The code block below is using both code and center classes.

.code {
  font-family: monospace;
  background-color: gray;
  display: block;
  position: relative;
  text-align: left;
}
.center {
    margin: auto;
    width: 30%;
    border: 3px solid navy;
    padding: 10px;
}
